TaskFlow – Manual Testing Case Study

A structured manual testing project conducted on a SaaS-based Task Management Web Application. The objective was to validate functional behavior, improve stability, and ensure production readiness through systematic test design and defect tracking.


Project Overview


  • Application Type: SaaS Web Application
  • Domain: Productivity / Task Management
  • Testing Scope: Functional, Regression, Validation
  • Methodology: Manual Testing (STLC Approach)
  • Tools Used: Excel (Test Cases), JIRA (Defect Tracking), Chrome

Test Strategy & Scope


  • Requirement analysis and scenario identification
  • Positive and negative test case design
  • Boundary value validation
  • Input field validation testing
  • Session and authentication validation
  • Regression coverage after defect fixes

Modules Covered


  • User Registration & Authentication
  • Dashboard Functionality
  • Create / Edit / Delete Task
  • Status Updates & Workflow Logic
  • Search & Filtering
  • User Profile Management
