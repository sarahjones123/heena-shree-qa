User Management REST API – Testing Case Study

This project demonstrates structured API testing practices applied to a REST-based User Management Service. Testing focused on request validation, response accuracy, status codes, authentication behavior, and data integrity.


Project Overview


  • API Type: RESTful Web Service
  • Format: JSON
  • Testing Scope: Functional, Negative, Validation
  • Tool Used: Postman
  • Methodology: API Test Case Design + Defect Logging

Endpoints Covered


  • POST /users – Create User
  • GET /users/{id} – Fetch User
  • PUT /users/{id} – Update User
  • DELETE /users/{id} – Delete User
  • GET /users – List All Users

Sample API Test Case


Test ID: API_TC_01

Scenario: Create user with valid payload

Request:


POST /users
{
  "name": "Heena",
  "email": "heena@test.com",
  "role": "admin"
}

Expected: Status 201 Created + User ID returned

Status: Pass

Negative Validation Case


Scenario: Missing email field


POST /users
{
  "name": "Heena",
  "role": "admin"
}

Expected: 400 Bad Request

Actual: 200 OK returned

Status: Fail

Sample Defect Report


Bug ID: API_002

Title: API returns 200 instead of 400 for missing mandatory field

Severity: High

Status: Open

Impact: Invalid data may be stored in database causing data inconsistency.
