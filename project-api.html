API Documentation Sample

Structured REST-style API documentation designed for developer-first clarity. This sample demonstrates endpoint definition, authentication, request/response modeling, and error handling best practices.

Overview

Base URL https://api.taskflow.com/v1
Protocol HTTPS
Response Format JSON

Authentication

All requests require a Bearer token in the Authorization header.

  Authorization: Bearer <your_access_token>

Tokens are generated using OAuth 2.0 authentication flow.

Endpoint

GET /projects

Retrieves a list of active projects available to the authenticated user.

Sample Request

  GET /projects HTTP/1.1
  Host: api.taskflow.com
  Authorization: Bearer <your_access_token>

Sample cURL

  curl -X GET "https://api.taskflow.com/v1/projects" \
    -H "Authorization: Bearer YOUR_ACCESS_TOKEN"

Sample Response

  {
    "projects": [
      {
        "id": "12345",
        "name": "Website Redesign",
        "status": "Active",
        "created_at": "2026-01-15T10:30:00Z"
      }
    ]
  }

Response Fields

  • id – Unique project identifier
  • name – Project title
  • status – Current project status
  • created_at – ISO 8601 timestamp

Error Handling

The API uses conventional HTTP status codes.

  {
    "error": {
      "code": 401,
      "message": "Unauthorized – Invalid or expired token"
    }
  }
